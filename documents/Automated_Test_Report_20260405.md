## 二、测试环境搭建

### 2.1 依赖安装

```bash
npm install --save-dev jest@27.0.4
```

### 2.2 测试脚本配置

在 `package.json` 中添加：

```json
{
  "scripts": {
    "test:web": "jest",
    "test:mp-weixin": "jest"
  }
}
```

### 2.3 Mock 配置

由于 `program` 对象由 uni-automator 在测试运行时注入，在普通 Jest 环境中需要 Mock 实现：

```javascript
// jest.setup.js
if (typeof program === 'undefined') {
  global.program = {
    reLaunch: async (url) => ({ path: url, query: {} }),
    navigateTo: async (url) => ({ path: url, query: {} }),
    currentPage: async () => ({ path: '/pages/index/index', query: {} }),
    pageStack: async () => []
  };
}
```

## 七、已知限制与改进建议

### 7.1 当前限制

1. **Mock 环境限制**: 当前使用 Mock 的 `program` 对象，无法完全模拟真实设备行为
2. **文本验证简化**: 部分文本内容验证被简化为元素存在性检查
3. **异步等待时间**: 使用固定等待时间，可能导致测试不稳定

### 7.2 改进建议

1. **增加集成测试**: 添加端到端测试，验证完整用户流程
2. **完善 Mock 数据**: 提供更丰富的 Mock 数据覆盖更多场景
3. **添加性能测试**: 验证页面加载时间和渲染性能
4. **增加错误处理测试**: 覆盖网络错误、数据异常等场景
5. **使用 data-testid**: 为关键元素添加 `data-testid` 属性，提高选择器稳定性
6. **CI/CD 集成**: 将测试集成到持续集成流程中
